# 06 - Technical Refinement

## Purpose

Select the technical direction after comparing realistic implementation options and trade-offs.

## Step-by-Step Workflow

1. Define candidate approaches.
2. Evaluate each approach for complexity, security, scalability, cost, latency, reliability, maintainability, and backward compatibility.
3. Choose one approach and explain why.
4. Record rejected approaches and follow-up questions.

- only one option was considered when multiple realistic options exist
- trade-offs are hidden behind preferences
- the chosen approach has no explicit reasoning
- open technical questions remain ownerless
